## 1. Diagram parser

type: feature | priority: p1 | depends: none | status: open

### Criteria

- `DiagramParser` implements `Parser` interface (Parse + Detect)
- Detects .mermaid, .mmd extensions (Mermaid)
- Detects .dot, .gv extensions (Graphviz DOT)
- Returns single Block with ContentType BlockContentDiagram
- Content preserved as-is (rendering happens in formatter layer)
- Stdin auto-detection: content starting with `graph`, `flowchart`, `sequenceDiagram`, `classDiagram`, `erDiagram`, `gantt`, `pie`, `gitgraph` detected as Mermaid
- Stdin auto-detection: content starting with `digraph` or `graph {` detected as DOT

- [ ] Implement Detect() for .mermaid, .mmd, .dot, .gv
- [ ] Implement Parse() returning raw content as single block
- [ ] Add BlockContentDiagram and BlockContentDOT to content_type.go
- [ ] Register in detectParser() in main.go
- [ ] Add diagram types to fileTypes map
- [ ] Add stdin detection in detectParserFromContent()

## 3. Web rendering via mermaid.js

type: feature | priority: p1 | depends: 1 | status: open

### Criteria

- `aster graph.mermaid --port 3000` renders the diagram as an SVG in the browser
- Mermaid.js loaded from CDN in --port mode
- Diagram renders client-side (mermaid.initialize + mermaid.run)
- Supports all Mermaid diagram types: flowchart, sequence, class, ER, gantt, pie, git
- Pan and zoom on the rendered SVG (click-drag to pan, scroll to zoom)
- Source code shown below diagram in collapsible code block
- SSE live reload works: edit .mermaid file, diagram re-renders

## 6. Markdown code fence integration
